# Pizza-Sales-Analysis-using-SQL-MariaDB-
I built a SQL project using MySQL/MariaDB on Linux (WSL) to analyze a pizza sales dataset containing around 48,630 records and 12 columns. I imported the CSV file using LOAD DATA INFILE, created the database and table, explored the dataset using SHOW, DESCRIBE, and SELECT statements, and converted the order_date column to the proper date format using STR_TO_DATE(). I used aggregate functions (SUM, COUNT, AVG, MIN, MAX), GROUP BY, HAVING, ORDER BY, subqueries, CTEs, and window functions such as RANK(), DENSE_RANK(), and LAG() to analyze sales performance, revenue trends, product performance, quarterly and monthly sales, peak ordering hours, average order value, category-wise revenue, and order patterns. I documented the SQL scripts and query outputs on GitHub.
